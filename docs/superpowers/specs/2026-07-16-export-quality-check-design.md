# ComicTranslator Export Quality Check Design

**วันที่:** 2026-07-16  
**สถานะ:** รอตรวจสเปกก่อน implementation

## เป้าหมาย

เพิ่มระบบตรวจคุณภาพทั้งตอนก่อน Export เพื่อแจ้งงานที่อาจยังไม่สมบูรณ์ ช่วยให้ผู้ใช้คลิกกลับไปแก้จุดที่พบได้ทันที แต่ไม่บล็อกการส่งออก ผู้ใช้ยังเลือกส่งออกต่อพร้อมคำเตือนได้เสมอ

## หลักการ

- ระบบ QA เป็นขั้นก่อน Export ไม่เปลี่ยน logic สร้างภาพ ไฟล์ หรือ ZIP เดิม
- คำเตือนต้องอธิบายหน้า กล่อง และสาเหตุที่แก้ได้
- ผลตรวจแบ่งเป็น `ข้อผิดพลาด`, `คำเตือน` และ `ผ่าน`
- คำเตือนทั้งหมดเป็น non-blocking
- การตรวจต้องไม่เรียก AI Inpainting, Gemini หรือสร้างไฟล์ Export
- ใช้ข้อมูลที่บันทึกไว้ของทุกหน้าและ Canvas text measurement เท่านั้น

## ลำดับการใช้งาน

1. ผู้ใช้กด `ส่งออกการ์ตูนไทย`
2. Export dialog เปิดและเริ่มตรวจคุณภาพทุกหน้า
3. แสดง progress ระหว่างตรวจ โดยไม่ทำให้ UI ค้าง
4. เมื่อเสร็จ แสดงสรุปจำนวนข้อผิดพลาด คำเตือน หน้าที่ผ่าน และหน้าที่ถูกยกเว้น
5. ผู้ใช้เลือกได้:
   - คลิกรายการเพื่อกระโดดไปแก้ไข
   - `ตรวจอีกครั้ง`
   - `กลับไปแก้ไข`
   - `ส่งออกต่อแม้มีคำเตือน`
6. หากไม่พบปัญหา ผู้ใช้เข้าสู่ตัวเลือก Export เดิมได้ทันที

## กฎการตรวจ

### 1. หน้าที่ยังไม่แปล

เตือนเมื่อไม่มีไฟล์ translation หรือข้อมูล translation ของหน้าว่าง และหน้านั้นไม่ได้ถูกทำเครื่องหมายว่า `ไม่ต้องแปล`

ผู้ใช้สามารถเลือก `หน้านี้ไม่ต้องแปล` จากรายการคำเตือน ค่านี้ยืนยันว่าเป็นหน้าภาพล้วนหรือไม่มีข้อความที่ต้องแปล และต้องถูกบันทึกต่อโปรเจกต์/ตอน/ชื่อหน้าเพื่อไม่ให้เตือนในการตรวจครั้งถัดไป

ผู้ใช้สามารถยกเลิกสถานะ `ไม่ต้องแปล` ได้จากผลตรวจ

### 2. กล่องข้อความว่าง

เตือนเมื่อกล่องที่ไม่ซ่อนมี `translated_text` ซึ่งว่างหรือมีเพียง whitespace

ไม่เตือนกล่องที่ `hidden === true` เพราะกล่องซ่อนตั้งใจใช้สำหรับรีทัชโดยไม่วาดข้อความไทย

### 3. กล่องหลุดนอกภาพหรือขนาดผิดปกติ

ตรวจทุกกล่อง รวมกล่องซ่อน เพราะพิกัดยังใช้ AI Inpainting หรือ Smooth Erase

ถือว่าผิดเมื่อ:

- `box_2d` ไม่มีตัวเลข finite ครบ 4 ค่า
- `ymin >= ymax` หรือ `xmin >= xmax`
- ค่าบางส่วนต่ำกว่า 0 หรือสูงกว่า 1000
- พื้นที่กล่องหลัง clamp เป็นศูนย์

### 4. ข้อความไทยล้นกรอบ

ตรวจเฉพาะกล่องที่ไม่ซ่อนและมีข้อความแปล

ใช้ Canvas `measureText()` พร้อม font family, font size, outline และขนาดกล่องเดียวกับเส้นทาง Export คำนวณการตัดบรรทัดตาม helper เดียวกับการวาดจริง ถือว่าล้นเมื่อจำนวนบรรทัดคูณ line height สูงกว่าพื้นที่แนวตั้งภายในกรอบ หรือมีคำหนึ่งคำที่กว้างเกินพื้นที่แนวนอนและไม่สามารถตัดได้ตามกฎ renderer

การตรวจต้องใช้ขนาดภาพจริงของแต่ละหน้าเพื่อแปลง `box_2d` จากสเกล 0–1000 เป็นพิกเซล

### 5. ชื่อเฉพาะไม่ตรง Glossary

สำหรับแต่ละคู่ `source → thai` ใน Project Glossary:

- ค้นหา `source` ใน `original_text` แบบไม่สนตัวพิมพ์ใหญ่–เล็ก
- ใช้ขอบเขตคำที่รองรับช่องว่างและเครื่องหมายวรรคตอน เพื่อไม่จับส่วนหนึ่งของคำอื่นโดยไม่ตั้งใจ
- เมื่อพบ source แต่ `translated_text` ไม่พบค่า Thai ที่กำหนด ให้สร้างคำเตือน
- ไม่ตรวจกล่องซ่อน
- รายการเดียวกันในกล่องเดียวกันเตือนเพียงครั้งเดียว

## ระดับผลตรวจ

`ข้อผิดพลาด` ใช้กับปัญหาที่ทำให้ผลภาพผิดชัดเจน:

- กล่องพิกัดไม่ถูกต้องหรือหลุดนอกภาพ
- ข้อความไทยล้นกรอบ

`คำเตือน` ใช้กับงานที่อาจตั้งใจหรือยังไม่เสร็จ:

- หน้ายังไม่แปล
- กล่องข้อความว่าง
- ชื่อเฉพาะไม่ตรง Glossary

ไม่มีระดับใดบล็อก Export

## โครงสร้างข้อมูลผลตรวจ

โมดูลตรวจคืนผลแบบ pure data:

```js
{
  pageName: '001.jpg',
  pageIndex: 0,
  status: 'error' | 'warning' | 'pass' | 'excluded',
  issues: [{
    code: 'PAGE_UNTRANSLATED' | 'EMPTY_TEXT' | 'INVALID_BOX' | 'TEXT_OVERFLOW' | 'GLOSSARY_MISMATCH',
    severity: 'error' | 'warning',
    bubbleId: null | string | number,
    message: 'ข้อความภาษาไทยสำหรับแสดงผล',
    details: {}
  }]
}
```

ตัวตรวจหลักต้องเป็นโมดูลที่ทดสอบด้วย Node.js ได้โดยไม่ต้องเปิด Electron ส่วนการโหลดภาพและสร้าง Canvas measurement adapter อยู่ที่ renderer

## UI ใน Export dialog

เพิ่มขั้น `ตรวจคุณภาพ` ก่อนตัวเลือก Export เดิม โดยมี:

- สรุปจำนวนข้อผิดพลาด คำเตือน ผ่าน และยกเว้น
- filter: `ทั้งหมด`, `ข้อผิดพลาด`, `คำเตือน`, `ผ่าน`
- รายการจัดกลุ่มตามหน้า แล้วเรียงตามลำดับหน้าและลำดับกล่อง
- ปุ่ม `หน้านี้ไม่ต้องแปล` เฉพาะคำเตือนหน้าที่ยังไม่แปล
- ปุ่ม `ตรวจอีกครั้ง`
- ปุ่ม `กลับไปแก้ไข`
- ปุ่ม `ส่งออกต่อแม้มีคำเตือน`
- เมื่อไม่มีปัญหา ปุ่มหลักใช้ข้อความ `ดำเนินการส่งออก`

ระหว่างตรวจให้แสดง progress `กำลังตรวจหน้า X/Y` และปิดปุ่มตรวจซ้ำเพื่อป้องกันงานซ้อน

## การกระโดดไปแก้ไข

เมื่อคลิก issue:

1. ปิด Export dialog
2. เปิดหน้าตาม `pageIndex` ผ่านเส้นทางเลือกหน้าเดิม
3. รอ translation และ image ของหน้านั้นโหลดเสร็จ
4. ถ้ามี `bubbleId` ให้เลือกกล่องนั้นทั้งบน Canvas และ Inspector
5. เลื่อน Inspector ให้การ์ดกล่องอยู่ในมุมมอง
6. รักษา render guard เดิมเพื่อไม่ให้ผลจากหน้าก่อนลอยมาหน้าใหม่

ถ้ากล่องถูกลบไปแล้วก่อนกระโดด ให้เปิดหน้าและแจ้งว่าไม่พบกล่อง โดยไม่ throw

## การบันทึกหน้าที่ไม่ต้องแปล

เก็บรายการชื่อหน้าที่ถูกยกเว้นใน metadata ของตอนผ่าน IPC ที่จำกัดขอบเขต project/chapter เดิม การเขียนต้องใช้ atomic JSON helper ที่มีอยู่ และไม่แก้ไฟล์ translation ของหน้า

ชื่อหน้าต้องตรวจสอบกับรายการภาพของตอนเพื่อป้องกัน path traversal และข้อมูลค้างจากไฟล์ที่ถูกลบ

## Error handling

- โหลด translation หน้าใดล้มเหลว: แสดง `ตรวจหน้าไม่สำเร็จ` เป็นคำเตือน และให้ตรวจซ้ำได้
- โหลดขนาดภาพล้มเหลว: ข้ามเฉพาะ TEXT_OVERFLOW แต่ยังตรวจกฎข้อมูลอื่น พร้อมเตือนว่าตรวจการล้นไม่ได้
- Canvas measurement adapter ล้มเหลว: ไม่บล็อก Export และไม่รายงานว่า “ผ่าน” สำหรับกฎที่ตรวจไม่ได้
- การบันทึกสถานะไม่ต้องแปลล้มเหลว: คืน checkbox เป็นค่าเดิมและแสดงข้อความผิดพลาด

## การทดสอบ

เพิ่ม unit tests สำหรับ:

- หน้าว่างและหน้าที่ถูกยกเว้น
- กล่องปกติ/ซ่อนที่ข้อความว่าง
- พิกัดครบ ขาด สลับด้าน และเกินขอบเขต
- ข้อความพอดีและล้นด้วย measurement adapter แบบ deterministic
- Glossary ตรง ไม่ตรง ตัวพิมพ์ต่างกัน เครื่องหมายวรรคตอน และ substring ที่ไม่ควรจับ
- การจัดระดับและลำดับ issue
- การรวมสรุปทั้งตอน
- การกระโดดไปหน้า/กล่องผ่าน controller ที่ทดสอบแยกได้
- UI contract ของ Export dialog และปุ่มส่งออกต่อ
- persistence ของหน้าที่ไม่ต้องแปลผ่าน atomic JSON

รัน `npm test` ทั้งชุดและ smoke test ใน Electron ด้วยโครงการจริง โดยไม่เรียก AI ระหว่างการสแกน

## เกณฑ์สำเร็จ

- ตรวจครบทั้งตอนก่อน Export โดยไม่สร้างไฟล์และไม่เรียก AI
- กล่องซ่อนยังรีทัชได้และไม่เกิดคำเตือนข้อความว่าง/ล้น/Glossary
- คลิก issue แล้วเปิดหน้าและเลือกกล่องถูกต้อง
- หน้าที่ทำเครื่องหมายไม่ต้องแปลไม่เตือนซ้ำและยกเลิกสถานะได้
- ผู้ใช้ส่งออกต่อได้เสมอแม้มีคำเตือน
- Export เดิมและ Facebook ZIP ยังทำงานเหมือนเดิม
- ชุดทดสอบเดิมและชุดใหม่ผ่านทั้งหมด
