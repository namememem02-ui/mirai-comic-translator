# Responsive Bubble Resize and Fit Modes Design

วันที่: 2026-07-15

## เป้าหมาย

แก้กรอบข้อความที่หดไม่ได้บนภาพมังฮวาแนวตั้งยาว และเพิ่มโหมดแสดงผลที่เหมาะกับทั้งมังฮวาและมังงะแบบหน้าเดียว โดยไม่กระทบพิกัด overlay, canvas, brush และระบบซูมเดิม

## สาเหตุของบั๊ก

ข้อมูล `box_2d` ใช้พิกัด normalized 0–1000 แต่ resize กำหนดความกว้างและความสูงขั้นต่ำเท่ากันที่ 20 หน่วย สำหรับภาพ `800 × 9500` ความสูงขั้นต่ำ 20 หน่วยเท่ากับประมาณ 190 พิกเซลจริง จึงไม่สามารถหดกรอบข้อความให้พอดีกับข้อความขนาดเล็กได้

## การแก้ Resize

- แปลงขนาดขั้นต่ำบนหน้าจอเป็นหน่วย SVG แยกแกน X และ Y
- ใช้ขั้นต่ำประมาณ 16 พิกเซลบนหน้าจอ ไม่ใช้ normalized 20 คงที่
- สูตร resize รับ `minWidth` และ `minHeight` แยกกัน
- รักษาขอบเขต 0–1000 และไม่ให้กรอบกลับด้าน
- handle เป็นสี่เหลี่ยมขนาดคงที่บนหน้าจอและคำนวณใหม่เมื่อ zoom/mode เปลี่ยน
- เพิ่ม regression test ด้วยอัตราส่วนภาพมังฮวา `800 × 9500`

## โหมดมุมมอง

### พอดีกว้าง

- ปรับความกว้างภาพให้พอดีกับพื้นที่ canvas
- รักษาอัตราส่วนภาพและเลื่อนลงเพื่ออ่านภาพยาว
- เหมาะกับมังฮวาและการ์ตูนจีน

### เต็มภาพ

- คำนวณ scale จากทั้งความกว้างและความสูงที่ว่าง แล้วเลือกค่าที่ทำให้เห็นภาพครบทั้งหน้า
- จัดภาพให้อยู่กลางพื้นที่ทำงาน
- เหมาะกับมังงะหรือหน้าการ์ตูนอัตราส่วนปกติ

### ซูมกำหนดเอง

- Slider และ Ctrl+Wheel ยังทำงานเหมือนเดิม
- เมื่อผู้ใช้ปรับ slider หรือ Ctrl+Wheel ระบบเปลี่ยนเป็นโหมดกำหนดเอง
- ปุ่ม Reset กลับไปโหมดพอดีกว้าง
- label แสดงเปอร์เซ็นต์จริงของมุมมองปัจจุบัน

## ความสอดคล้องของ Layer

`viewportContainer` เป็นตัวกำหนดขนาดร่วมของภาพ, SVG overlay, typeset canvas, mask canvas และ paint canvas ทุก layer จึงต้องเปลี่ยนขนาดผ่าน container เดียว ห้ามใช้ transform เฉพาะภาพ

หลังเปลี่ยนขนาด ต้องเรียกอัปเดต handle เพื่อรักษาขนาดบนหน้าจอ และตรวจว่า coordinate conversion ยังอิง bounding rectangle ของ overlay

## การทดสอบ

- ลากมุมขวาล่างไปซ้าย/ขึ้นแล้วกรอบหดได้ต่ำกว่า 190 พิกเซลบนภาพ `800 × 9500`
- minimum size บนหน้าจอใกล้เคียง 16 พิกเซลทั้งแกน X/Y
- พอดีกว้างทำให้ความกว้างภาพไม่เกิน canvas
- เต็มภาพทำให้ทั้งความกว้างและความสูงไม่เกิน canvas
- Slider เปลี่ยนเป็น custom mode และ overlay ตรงกับภาพ
- รัน unit tests และ syntax check

## นอกขอบเขต

- ไม่เปลี่ยนรูปแบบข้อมูล `box_2d`
- ไม่เพิ่ม handle หลายมุมในรอบนี้
- ไม่เปลี่ยน logic การวาดข้อความหรือ AI OCR
